mappingBuilder - append

append ( path )

The specified path in the model is appended to (referenced by) the viewmodel as is. The object and it's children will not be made observable. The path's properties/children will not processed by default or custom mapping.

Note: Appended paths that contain strings, numbers, dates, nulls and undefined values are evectively copied, but if the appended path refers to an object then changes to the properties of the object in the viewmodel will also change those properties in the model (because both the model and viewmodel contain references to the same object). This is normally not a concern unless the original model needs to be preserved for some reason, such as for a dirty comparison or to revert changes.

Parameters:

path - either a single path as a string or an array of paths.

Example - Appending a single path

var mappingOptions = ko.viewmodel.mappingBuilder().append("{root}.users[i]");

var viewmodel = ko.viewmodel.fromModel(model, mappingOptions);
Example - Appending multiple paths with multiple calls

var mappingOptions = ko.viewmodel.mappingBuilder()
	.append("{root}.users[i]")
	.append("{root}.currentUser");
	
var viewmodel = ko.viewmodel.fromModel(model, mappingOptions);
Example - Appending multiple paths by passing in an array of paths:

var mappingOptions = ko.viewmodel.mappingBuilder().append([
	"{root}.users[i]",
	"{root}.currentUser",
]);

var viewmodel = ko.viewmodel.fromModel(model, mappingOptions);
