mappingBuilder - exclude

exclude ( path )

The specified path in the model is excluded from the viewmodel. The path and it's children will not be in the viewmodel and will not be a part of models generated by calling viewmodel.toModel on the viewmodel.

Note: This method is used to exclude data that is not needed by the view and does not need to be returned to the server. If the data does need to be returned to the server, but doesn't need to be observable then use append instead.

Parameters:

path - either a single path as a string or an array of paths.

Example - excluding a single path

var mappingOptions = ko.viewmodel.mappingBuilder().exclude("{root}.users");

var viewmodel = ko.viewmodel.fromModel(model, mappingOptions);
Example - excluding multiple paths with multiple calls

var mappingOptions = ko.viewmodel.mappingBuilder()
	.exclude("{root}.users")
	.exclude("{root}.currentUser");
	
var viewmodel = ko.viewmodel.fromModel(model, mappingOptions);
Example - excluding multiple paths by passing in an array of paths:

var mappingOptions = ko.viewmodel.mappingBuilder().exclude([
	"{root}.users",
	"{root}.currentUser",
]);

var viewmodel = ko.viewmodel.fromModel(model, mappingOptions);
