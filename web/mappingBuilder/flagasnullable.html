mappingBuilder - FlagAsNullable

flagAsNullable ( path )

The method flags the specified path as a nullable object. Default processing for nulls and objects are different. Objects aren't made observable for performance reasons. Null values are always observable, but have no children to process. This cause two problems:

First, if the path is initially an object then changing it to a null will have no effect on the UI. This is because there is no observable to trigger a response to the change. Flag as null instructs the viewmodel plugin to always wrap objects in null.

Second, if the path is initially a null that is updated to an object then that object will never recieve processing on it's children.  Flagging the path as nullable instructs the viewmodel plugin to do full processing on the object whenever the previous value was a null.

Note: For the purposes of this method both null and undefined are treated as equivalent.

Parameters:

path - either a single path as a string or an array of paths.

Example - Flagging a single path as nullable

var mappingOptions = ko.viewmodel.mappingBuilder()
	.flagAsNullable("{root}.users[i].settings");

var viewmodel = ko.viewmodel.fromModel(model, mappingOptions);
Example - Appending multiple paths with multiple calls

var mappingOptions = ko.viewmodel.mappingBuilder()
	.flagAsNullable("{root}.users[i].settings")
	.flagAsNullable("{root}.currentUser");
	
var viewmodel = ko.viewmodel.fromModel(model, mappingOptions);
Example - Appending multiple paths by passing in an array of paths:

var mappingOptions = ko.viewmodel.mappingBuilder().flagAsNullable ([
	"{root}.users[i].settings",
	"{root}.currentUser",
]);

var viewmodel = ko.viewmodel.fromModel(model, mappingOptions);
